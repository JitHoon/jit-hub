## 1. Obsidian Publish — 그래프 뷰 미학

**차용할 패턴:**

- **별자리 테마**: 다크 배경 `#161616` + 보라-다크 그라디언트 오버레이, `background-blend-mode: soft-light`로 깊이감
- **노드 색상 시스템**: CSS 변수로 node fill, border, connection line, hover highlight, focused 상태 분리 — 우리 클러스터 색상 시스템과 1:1 매핑 가능
- **Canvas 글로우 구현**: Canvas 기반 렌더링에서 글로우 효과는 CSS가 아닌 Canvas API의 `shadowBlur`, `shadowColor` 사용 필요 (react-force-graph-2d도 동일)

---

## 2. Linear.app — 분할 패널 UI, 다크 모드

**차용할 패턴:**

- **LCH 색상 공간 검증**: Linear도 HSL → LCH 전환. 핵심 변수 3개 (base, accent, contrast) — 우리 OKLCH 접근법 정당성 확인
- **따뜻한 회색 다크 모드**: 차가운 블루 → 따뜻한 그레이로 전환. "구조는 느껴져야지 보여선 안 된다"
- **사이드바 깊이 계층**: 사이드바(그래프)를 메인 콘텐츠보다 "몇 단계 어둡게" → 시각적 깊이
- **트랜지션 값**: `transition: 0.2s opacity ease-out, transform ease-out 0.2s`, hover `scale(1.01) translateY(-4px)`, `box-shadow: 0 10px 40px`

## 3. The Pudding — 데이터 시각화 품질

**차용할 패턴:**

- **메시지 우선 시각화**: "데이터를 시각화하기 전에 전달할 핵심 포인트를 먼저 정리하라" — 그래프 노드는 클러스터 소속과 중요도를 한눈에 전달해야 함
- **타이포그래피 삼중 구조**: 디스플레이(제목) + 본문 + UI 폰트 분리. Noto Sans KR 700(제목) + 400(본문) + 500(UI) 매핑 가능

---

## 4. Raycast — 다크 모드 퍼스트 컴포넌트

**차용할 패턴:**

- **자동 색상 적응**: 라이트/다크 테마별 대비 자동 조정 — 9개 OKLCH 클러스터 색상의 모드별 가독성 보장에 참고
- **Detail 컴포넌트 패턴**: 리치 콘텐츠 + 메타데이터 조합 — 우리 콘텐츠 패널(제목 + 배지 + MDX + 관련 링크)과 동일 구조

---

## 5. Excalidraw — 캔버스 + 패널 분할

**차용할 패턴:**

- **다크 테마 토큰 (실전 검증)**: Background `#121212`, Island/패널 `#232329`, Surface high `#2e2d39`, Border `#46464f` — 배틀 테스트된 다크 색상 세트
- **Island 컴포넌트**: 캔버스 위 플로팅 UI 컨테이너 — 그래프 컨트롤 오버레이에 적용 가능
- **Dockable 사이드바 + 브레이크포인트**: 컨테이너 너비 초과 시 사이드바가 도킹되어 "UI를 좌측으로 밀어냄" — 우리 분할 뷰의 반응형 패턴과 정확히 일치
- **Z-index 계층 시스템**: CSS 변수로 canvas, interactive layer, modal, popover 충돌 방지

---

## 6. Stripe Docs — 콘텐츠 패널 타이포그래피

**차용할 패턴:**

- **8px 기반 간격 스케일**: 0/2/4/8/16/24/32/48px — 일관된 간격 시스템
- **비율 기반 너비**: 1/2, 1/3, 2/3, 1/4, 3/4 — 고정 px 대신 비율로 반응형 분할
- **콘텐츠 계층 패턴**: 개요 → 비교 → 커스터마이징 → 코드 예제 순서 — 우리 노드 콘텐츠(제목 → 요약 → 상세 → 관련 노드)에 매핑
- **선언적 타이포그래피 토큰**: px 대신 heading/body/caption 시맨틱 토큰

---

## 7. Neo4j Bloom — 그래프 시각화 인터랙션

**차용할 패턴:**

- **범례 패널**: 카테고리별 색상 + 클릭으로 스타일 변경 — 클러스터 필터/범례로 적용
- **그룹 노드 = 큰 노드**: 그룹화된 노드는 "시각적으로 구별되는 큰 노드" — 허브 노드 표현에 직접 적용
- **필터 = 회색 처리**: 필터된 요소를 숨기지 않고 회색으로 표시 — 발견 가능성 유지
- **줌 반응형 텍스트**: 줌 레벨에 따라 노드 텍스트 크기 자동 조정
- **호버 툴팁**: 설정 가능한 속성 표시 — 노드 호버 시 제목 + 클러스터 + 난이도 표시

## 다크 모드 데이터 시각화 가이드라인

- **배경**: `#1E1E1E` ~ `#2D2D2D`, 순수 검정 `#000000` 금지
- **텍스트 계층**: Primary `#FFFFFF`, Labels `#E0E0E0`, Secondary `#B0B0B0`
- **데이터 색상**: 라이트 모드 대비 채도 낮추고 명도 20~30% 높이기
- **선 두께**: 다크 모드에서 2~3px (라이트 1.5~2px)
- **폰트 크기**: 다크 모드에서 1pt 정도 키우기 (대비 낮아 가독성 보완)
- **OKLCH 다크 팔레트**: 배경 shade 900 (명도 12%), 텍스트 shade 200 (명도 89%), 악센트 shade 300 (명도 80%)
