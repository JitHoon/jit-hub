## 테스트 & 품질

**현실**: 역삼각형 피라미드 (E2E 7 > Unit 5, Integration 0). 현재 규모에서는 작동하나 확장 시 위험.

- E2E가 실제 버그 3건+ 발견 (WebGL 폴백, URL 파라미터, 테마 동기화)
- 유닛 테스트 밀도 극히 낮음 (1 테스트/929 LOC)
- 통합 테스트 0건 — 콘텐츠 파이프라인 전체 흐름 미검증
- 접근성: WCAG A 수준 (AA 미달), 3D 키보드 탐색/prefers-reduced-motion 미지원
- 성능: three.js ~150KB gzip, Lighthouse 베이스라인 미측정

## 7가지 핵심 원칙

1. **요청 범위만 수행** — 과잉 구현 금지
2. **점진적 복잡도** — 단순한 것부터, 필요할 때 확장
3. **빌드 타임 검증** — 잘못된 데이터가 배포되지 않도록
4. **하네스 = 프롬프트** — 시스템 레벨에서 AI 행동을 가이드
5. **What은 인간, How는 AI** — 위임 경계 명확히
6. **문서는 Why만** — 코드가 설명하는 것은 문서화 금지
7. **첫 주 초반에 핵심 결정 집중** — Day 1~3에 아키텍처 확정
