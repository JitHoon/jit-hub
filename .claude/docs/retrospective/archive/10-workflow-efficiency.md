## 워크플로우 패턴

### 효과적이었던 패턴

1. **Phase 기반 점진적 개발**
   - progress.md로 태스크 추적
   - Phase별 PR 생성으로 리뷰 단위 관리
   - 완료 시점이 명확

2. **작은 PR + 빈번한 머지**
   - 일평균 4 PR → 피드백 루프 짧음
   - 충돌 최소화
   - 각 PR이 독립적으로 리버트 가능

3. **하네스 기반 자동화**
   - PostToolUse 훅: 코드 편집 후 즉시 TypeScript/ESLint 검사
   - PreToolUse 훅: main 브랜치 보호, 시크릿 감지
   - 수동 검증 시간 절약

### 병목이었던 구간

1. **hydration 버그 디버깅** (4커밋, 추정 ~2시간)
   - SSR/CSR 차이 이해 부족이 원인
   - 사전에 library-practices에 패턴 문서화했으면 방지 가능

2. **OG 이미지 디자인** (반복 수정)
   - 시각적 결과물은 AI 한계
   - 빠른 프리뷰 루프 도구 필요

3. **v1→v2 블루프린트 전환** (1일 소요)
   - 초기 설계 검증 부족이 원인
   - Day 1에 MVP 스코프를 더 명확히 했으면 단축 가능

## 하네스 관련 작업 비율

| 구분 | 추정 커밋 수 | 비율 |
|------|-------------|------|
| 하네스 구축/수정 | ~40 | ~16% |
| 기능 구현 | ~120 | ~49% |
| 버그 수정/리팩토링 | ~50 | ~21% |
| 인프라/배포/설정 | ~33 | ~14% |

하네스에 전체 작업의 ~16%를 투자. 이 투자가 나머지 84%의 품질과 속도에 기여.

## 시간 효율성 지표

| 지표 | 값 | 평가 |
|------|------|------|
| 기능 당 평균 구현 시간 | ~0.5일 | AI 협업 효과 |
| 버그 수정 평균 시간 | ~1시간 | PostToolUse 훅 효과 |
| PR 머지까지 평균 시간 | <1시간 | 1인 프로젝트 + 자동 검증 |
| Phase 완료 평균 시간 | ~0.5일 | 잘게 나눈 Phase 효과 |

## 개선 기회

1. **Day 1 프로젝트 스캐폴딩 자동화**: 초기화 작업(설정 파일, CI, 하네스 기본 구조)을 템플릿으로
2. **blueprint 검증 체크리스트**: v1→v2 전환 같은 대규모 변경을 사전에 방지
3. **시각 결과물 프리뷰 루프**: OG 이미지 등 디자인 작업의 빠른 피드백
4. **병렬 작업 기회 활용**: 독립적인 Phase를 동시에 진행
