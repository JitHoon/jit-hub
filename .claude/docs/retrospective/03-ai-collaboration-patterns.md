# 03. AI 협업 패턴 분석

## 개요

7일간 243커밋(일 평균 35커밋)이라는 높은 생산성을 AI 협업으로 달성했다. 어떤 패턴이 효과적이었고, 어떤 것이 비효율적이었는지 분석한다.

## 생산성 지표

| 지표 | 값 | 의미 |
|------|------|------|
| 일 평균 커밋 | ~35 | 전통적 개인 프로젝트의 5~10배 수준 |
| 일 평균 PR | ~4 | 잦은 피드백 루프 |
| 평균 커밋 크기 | ~19 LOC | 작은 단위의 원자적 변경 |
| 총 Phase | 15+ | 체계적 단계별 진행 |

## AI에게 위임하기 좋았던 작업 유형

### 높은 효과 (★★★★★)
1. **보일러플레이트 생성**: sitemap.ts, robots.ts, manifest, 메타데이터 설정
2. **패턴 반복**: 11개 아이콘 컴포넌트, 9개 클러스터 색상 정의
3. **설정 파일**: ESLint, Tailwind, TypeScript config 작성
4. **테스트 작성**: E2E 시나리오 구현, 유닛 테스트 케이스
5. **리팩토링**: inline style → Tailwind 클래스, 유틸리티 추출

### 보통 효과 (★★★☆☆)
1. **아키텍처 설계**: 방향 제시는 인간이, 구체화는 AI가
2. **버그 수정**: hydration 버그처럼 원인 파악은 협업, 수정은 AI
3. **코드 리뷰**: 패턴 위반 감지는 잘하나, 비즈니스 로직 판단은 제한적

### 낮은 효과 (직접 해야 하는 작업)
1. **제품 기획**: 어떤 노드를 만들지, 클러스터를 어떻게 나눌지는 도메인 전문성 필요
2. **UX 결정**: 분할 뷰 비율, 인터랙션 패턴, 시각적 우선순위
3. **콘텐츠 작성**: 3D GIS 전문 지식 기반 마크다운 노드 내용
4. **하네스 설계 철학**: 어떤 스킬이 필요한지, 오케스트레이션 패턴 결정

## 프롬프트 패턴 분석

### 효과적이었던 프롬프트 패턴
1. **구체적 파일 경로 + 변경 내용 명시**: "src/app/layout.tsx에서 메타데이터 추가"
2. **기존 패턴 참조**: "cluster.ts와 같은 방식으로 tokens.ts 생성"
3. **Phase 기반 요청**: progress.md의 태스크 참조하여 범위 명확화
4. **부정 조건 명시**: "인라인 스타일 사용하지 말고 Tailwind 클래스로"

### 비효율적이었던 프롬프트 패턴
1. **모호한 개선 요청**: "코드를 더 좋게 만들어줘" → 범위 불명확으로 과잉 수정
2. **한 번에 너무 많은 요청**: 여러 파일 동시 수정 → 누락 발생
3. **컨텍스트 부족한 버그 리포트**: "동작 안 해" → 재현 조건 불명확

## AI 협업에서 발견한 패턴

### 1. 하네스가 곧 프롬프트 엔지니어링
- CLAUDE.md의 절대 규칙, Skills의 Gotchas, Rules의 코드 스타일이 매번 프롬프트에 반복할 내용을 시스템 레벨에서 해결
- 프롬프트에 "TypeScript any 쓰지 마" 대신 룰로 한 번 정의하면 모든 세션에서 적용

### 2. 점진적 복잡도 증가
- 처음에는 간단한 파일 생성부터 시작
- 패턴이 확립되면 AI가 자율적으로 패턴을 따름
- 복잡한 기능(3D 그래프, 콘텐츠 파이프라인)은 단계적으로 구축

### 3. 검증 루프의 중요성
- PostToolUse 훅(TypeScript/ESLint 자동 검사)이 AI 코드 품질을 실시간 보장
- 검증 없이 대량 코드 생성 → 나중에 일괄 수정 필요 → 비효율

### 4. 위임의 경계
- "무엇을(What)" 만들지는 인간이 결정
- "어떻게(How)" 만들지는 AI에게 위임 가능 (단, 하네스로 가이드라인 제공)
- "왜(Why)" 그 결정을 했는지는 blueprint 문서로 기록

## 반복 수정이 많았던 영역

| 영역 | 반복 원인 | 교훈 |
|------|----------|------|
| hydration 버그 | 4개 커밋으로 점진 수정 | SSR/CSR 차이를 사전에 스킬 문서화 |
| 테마 전환 깜빡임 | useEffect → useLayoutEffect | React 18 패턴을 library-practices에 추가 |
| OG 이미지 | 스타일 수정 반복 | 시각적 결과물은 AI 한계 — 빠른 프리뷰 루프 필요 |
| 커밋 메시지 | conventional commits 미준수 | 룰 추가 후 개선 |
