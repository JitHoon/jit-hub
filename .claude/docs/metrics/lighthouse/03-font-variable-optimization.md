# Lighthouse 개선 #2: 폰트 가변(Variable) 전환

> 작업일: 2026-04-11
> 대상 카테고리: Performance (FCP/LCP 개선)

---

## 문제

`Noto_Sans_KR`과 `Lexend`를 각각 정적 3개 웨이트(400, 500, 700)로 로드.
유니코드 범위 슬라이스당 **웨이트별 별도 파일**이 필요하여 폰트 요청 수가 3배.

| 폰트 | 웨이트 | 슬라이스당 파일 수 |
|------|--------|------------------|
| Noto Sans KR | 400, 500, 700 (정적) | 3개 |
| Lexend | 400, 500, 700 (정적) | 3개 |

홈 페이지 초기 로드 시 한글 폰트 파일이 FCP/LCP를 지연.

---

## 변경 내용

`next/font/google`에서 명시적 `weight` 제거 → 가변(Variable) 폰트로 전환.
슬라이스당 1개 파일로 통합되어 폰트 요청 수 ~1/3로 감소.

| 폰트 | 변경 전 | 변경 후 | 슬라이스당 파일 수 |
|------|---------|---------|------------------|
| Noto Sans KR | `weight: ["400", "500", "700"]` | 가변 (weight 제거) | 1개 |
| Lexend | `weight: ["400", "500", "700"]` | 가변 (weight 제거) | 1개 |

### 추가 효과

- `font-semibold` (600)가 이전에는 700으로 폴백 → 이제 정확한 600 렌더링
- 모든 weight (100~900) 사용 가능

### 수정 파일

| 파일 | 변경 |
|------|------|
| `src/app/layout.tsx` | `Noto_Sans_KR`, `Lexend` weight 파라미터 제거 |
| `.storybook/preview-head.html` | Google Fonts URL을 가변 폰트 형식으로 변경 |
