# Lighthouse 용어 사전

Lighthouse가 측정하는 핵심 지표와 그 의미를 정리한 문서.

---

## Core Web Vitals (핵심 웹 지표)

Google이 사용자 경험을 측정하기 위해 정의한 3가지 핵심 지표. 검색 순위에도 영향.

### FCP — First Contentful Paint (첫 콘텐츠 표시)

페이지 로드 시작 후 **처음으로 텍스트나 이미지가 화면에 나타나는 시점**.

- 사용자가 "이 페이지가 로딩되고 있구나"를 처음 느끼는 순간
- 빠를수록 좋음: **0.8s 이하**면 우수, 1.8s 초과면 느림
- 영향 요인: 서버 응답 시간, 렌더링 차단 리소스(CSS/JS), 폰트 로딩

### LCP — Largest Contentful Paint (최대 콘텐츠 표시)

뷰포트에서 **가장 큰 콘텐츠 요소**(보통 메인 이미지나 제목 텍스트)가 완전히 렌더링되는 시점.

- 사용자가 "메인 콘텐츠가 다 로드됐다"고 느끼는 순간
- **1.2s 이하**면 우수, 2.5s 초과면 느림
- 영향 요인: 이미지 크기/포맷, 폰트 로딩, 서버 응답 시간, 렌더링 차단 JS

### CLS — Cumulative Layout Shift (누적 레이아웃 이동)

페이지 로딩 중 요소들이 **예상치 못하게 위치가 바뀌는 정도**를 수치화한 것.

- 버튼을 누르려는데 갑자기 광고가 삽입되어 위치가 밀리는 현상
- **0이면 완벽**, 0.1 초과면 개선 필요
- 영향 요인: 크기 미지정 이미지/iframe, 동적 삽입 콘텐츠, 웹폰트 교체(FOUT)

## 추가 성능 지표

### TBT — Total Blocking Time (총 차단 시간)

FCP와 TTI 사이에서 **메인 스레드가 50ms 이상 차단된 시간의 합계**.

- 이 시간 동안 사용자의 클릭/타이핑에 브라우저가 반응하지 못함
- **0ms**면 완벽, 200ms 초과면 개선 필요
- 영향 요인: 무거운 JavaScript 실행 (파싱, 컴파일, 평가)
- 예시: three.js 같은 대형 라이브러리를 한꺼번에 로드하면 TBT가 치솟음

### TTI — Time to Interactive (상호작용 가능 시점)

페이지가 **완전히 상호작용 가능해지는 시점**. 모든 이벤트 핸들러가 등록되고, 메인 스레드가 안정화된 상태.

- 사용자가 버튼을 눌렀을 때 즉시 반응하기 시작하는 시점
- **2.0s 이하**면 우수, 3.8s 초과면 느림
- 영향 요인: JavaScript 번들 크기, 서드파티 스크립트, 메인 스레드 작업량

### SI — Speed Index (속도 지수)

페이지 콘텐츠가 **시각적으로 채워지는 속도**를 종합 점수화한 것.

- 빈 화면 → 완성 화면까지 얼마나 빠르게 진행되는지 측정
- **1.0s 이하**면 우수, 3.4s 초과면 느림
- FCP/LCP와 달리 전체 렌더링 과정의 "부드러움"을 반영

## Lighthouse 4대 카테고리

### Performance (성능)

위 지표들을 가중 합산한 종합 점수. 각 지표의 가중치:

| 지표 | 가중치 |
|------|--------|
| TBT | 30% |
| LCP | 25% |
| CLS | 25% |
| SI | 10% |
| FCP | 10% |

### Accessibility (접근성)

장애가 있는 사용자도 웹을 이용할 수 있는지 평가. 주요 항목:
- **색상 대비**: 텍스트와 배경의 명암비 (WCAG AA 기준 4.5:1 이상)
- **시맨틱 HTML**: 올바른 heading 계층, landmark 역할
- **키보드 접근**: 모든 인터랙티브 요소에 키보드로 접근 가능
- **대체 텍스트**: 이미지에 alt 속성

### Best Practices (모범 사례)

보안, 최신 웹 표준 준수 여부. 주요 항목:
- HTTPS 사용
- 안전하지 않은 라이브러리 감지
- 콘솔 에러 없음
- 올바른 이미지 종횡비

### SEO (검색 엔진 최적화)

검색 엔진이 페이지를 잘 이해하고 색인할 수 있는지 평가. 주요 항목:
- meta title/description 존재
- 크롤링 가능한 링크
- 올바른 viewport 설정
- 구조화된 데이터 (JSON-LD)

## WCAG AA 색상 대비 기준

| 텍스트 유형 | 필요 대비비 |
|------------|-----------|
| 일반 텍스트 (< 18pt) | 4.5:1 이상 |
| 큰 텍스트 (>= 18pt 또는 14pt bold) | 3:1 이상 |
| UI 컴포넌트/그래픽 | 3:1 이상 |

대비비 = (밝은색 휘도 + 0.05) / (어두운색 휘도 + 0.05)
