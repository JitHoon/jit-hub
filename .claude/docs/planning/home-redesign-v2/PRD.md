# PRD: 홈페이지 리디자인 v2 (세로 스크롤 구조)

## 개요

- **배경**: 현재 홈페이지는 좌우 분할 뷰(그래프 38% + 콘텐츠 62%) 구조로 구성되어 있다. 이 레이아웃은 데스크톱에서는 괜찮지만, 모바일 대응이 복잡하고 그래프가 축소되면 탐색성이 떨어진다. 또한 네비게이션 구조가 없어 Projects 페이지로의 진입점이 부재하다.
- **목표**: 세로 스크롤 구조로 전환하여 그래프를 히어로 섹션으로 전면 배치하고, 노드 클릭 시 아래로 콘텐츠가 펼쳐지는 직관적인 플로우를 구현한다. 상단 헤더에 네비게이션을 추가하여 사이트 구조를 명확히 한다.
- **타겟 사용자**: 개발자 채용 담당자, 기술 블로그 방문자, 동료 개발자

## 사용자 스토리

- As a **채용 담당자**, I want 포트폴리오에 접속하자마자 풀스크린 그래프에서 기술 역량을 한눈에 파악하고 싶다, so that 면접 전에 후보자의 기술 수준을 빠르게 판단할 수 있다.
- As a **방문자**, I want 관심 있는 노드를 클릭하면 아래로 콘텐츠가 자연스럽게 펼쳐지길 원한다, so that 별도 페이지 이동 없이 콘텐츠를 탐색할 수 있다.
- As a **방문자**, I want 다른 노드를 클릭하면 본문 내용이 교체되고 본문 상단으로 스크롤되길 원한다, so that 여러 주제를 효율적으로 비교 탐색할 수 있다.
- As a **방문자**, I want 헤더의 Projects 링크를 통해 AI 프로젝트 목록을 확인하고 싶다, so that 포트폴리오의 다른 측면도 탐색할 수 있다.
- As a **모바일 사용자**, I want 데스크톱과 동일한 세로 스크롤 구조에서 그래프와 콘텐츠를 볼 수 있다, so that 어떤 기기에서든 일관된 경험을 얻을 수 있다.

## 기능 명세

### MVP (즉시 구현)

| 기능 | 설명 | 완료 기준 |
|------|------|----------|
| 상단 헤더 | sticky top 가로 바. 왼쪽: JIT-Hub 로고, 오른쪽: Projects 링크 + 테마 토글 | 스크롤해도 상단 고정, 로고/링크/토글 정상 렌더링 |
| 히어로 그래프 섹션 | 헤더 아래 나머지 뷰포트 전체 높이(`calc(100vh - 헤더높이)`)를 차지하는 그래프 영역 | 데스크톱/모바일 모두 동일하게 뷰포트 채움 |
| 노드 클릭 슬라이드 다운 | 노드 클릭 시 그래프 아래에 콘텐츠 영역이 슬라이드 다운 애니메이션으로 펼쳐짐 + 자동 스크롤 | 애니메이션 부드러움, 콘텐츠 영역으로 자동 스크롤 |
| 노드 전환 | 다른 노드 클릭 시 본문 내용 교체 + 본문 상단으로 스크롤 | 콘텐츠 전환 시 깜빡임 없이 교체, 상단으로 스크롤 |
| URL 동기화 | 선택된 노드가 `?node=slug` 쿼리 파라미터로 반영 | 새로고침 시 선택 상태 복원 + 콘텐츠 펼침 |
| 기존 ThemeToggle 이동 | fixed 위치에서 헤더 우측으로 이동 | 헤더 안에서 정상 동작 |
| SEO 경로 유지 | `/nodes/[slug]` 정적 페이지 유지 | 빌드 시 정적 페이지 생성 정상 |

### Phase 2 (검증 후)

| 기능 | 설명 | 조건 |
|------|------|------|
| `/projects` 페이지 | 외부 링크 카드 리스트 형태의 AI 프로젝트 페이지 | 헤더 내비게이션 완성 후 |
| 콘텐츠 접기 | 펼쳐진 콘텐츠를 접고 그래프만 보는 기능 | 사용자 피드백 후 |
| 노드 검색/필터 | 클러스터별 필터, 키워드 검색 | 노드 수 20개 이상 시 |

## 비기능 요구사항

- **성능**: 그래프 초기 렌더 2초 이내 (50개 노드 기준). 슬라이드 다운 애니메이션 60fps. LCP 3.0s 미만.
- **보안**: 사용자 입력 없음. XSS 위험 없음.
- **접근성**: 헤더 내비게이션 키보드 탐색 가능. 테마 토글 aria-label 유지. 콘텐츠 펼침/접기 시 포커스 관리. 그래프 캔버스 aria-label 제공.
- **SEO**: `/nodes/[slug]` 정적 페이지 유지. 홈의 `?node=slug`는 SPA 탐색용.

## 성공 지표 (KPI)

- **주관적 완성도**: 완성된 화면을 직접 보고 판단 [사용자 육안 확인]
- **레이아웃 일관성**: 데스크톱/모바일에서 동일한 세로 스크롤 구조 동작 [크로스 디바이스 테스트]
- **전환 부드러움**: 슬라이드 다운, 콘텐츠 교체 시 60fps 유지 [Performance 프로파일]

## 스코프 아웃

이 기능에서 명시적으로 다루지 않는 것:
- `/projects` 페이지 자체의 콘텐츠 구현 (헤더 링크만 추가, 페이지는 Phase 2)
- 그래프 라이브러리 변경 (현재 react-force-graph-3d 기반 유지)
- 노드 검색/필터 기능
- 다국어(i18n) 지원
- 좌우 분할 뷰 (완전 제거, 세로 스크롤로 대체)
- 댓글, 좋아요 등 소셜 기능
