# PRD: 홈페이지 리디자인 (Starfield Graph 3D)

## 개요

- **배경**: 현재 홈페이지(`/`)는 "JIT-Hub" 텍스트만 표시하는 플레이스홀더 상태다. 3D GIS 지식 포트폴리오로서 방문자에게 인상적인 첫인상을 주려면, 그래프 노드가 3D 공간에서 별처럼 빛나며 떠다니는 몰입형 랜딩 경험이 필요하다.
- **목표**: 포트폴리오 방문 시 "이 사람의 기술 역량이 한눈에 느껴지는" 3D 시각적 첫인상 제공. 3D 공간 탐색(회전, 줌, 드래그)과 노드 클릭으로 자연스럽게 콘텐츠 탐색까지 이어지는 원스텝 플로우 구현.
- **타겟 사용자**: 개발자 채용 담당자, 기술 블로그 방문자, 동료 개발자

## 사용자 스토리

- As a **채용 담당자**, I want 포트폴리오에 접속하자마자 3D 공간에 펼쳐진 기술 역량의 넓이와 깊이를 시각적으로 느끼고 싶다, so that 면접 전에 후보자의 기술 수준을 빠르게 파악할 수 있다.
- As a **방문자**, I want 3D 공간을 회전하고 줌인/줌아웃하며 노드를 탐색하고, 관심 있는 것을 클릭하면 관련 글이 자연스럽게 나타나길 원한다, so that 별도 페이지 이동 없이 탐색할 수 있다.
- As a **방문자**, I want 다크모드에서 노드가 3D 공간 속 별처럼 빛나는 경험을 하고 싶다, so that 포트폴리오가 기억에 남는다.
- As a **재방문자**, I want 네비게이션을 통해 Article/Project 페이지로 이동하고 싶다, so that 다른 형태의 콘텐츠도 탐색할 수 있다.
- As a **모바일 사용자**, I want 모바일에서도 3D 그래프와 콘텐츠를 볼 수 있다, so that 어디서든 포트폴리오를 확인할 수 있다.

## 기능 명세

### MVP (즉시 구현)

| 기능 | 설명 | 완료 기준 |
|------|------|----------|
| 풀스크린 3D 그래프 | 홈 진입 시 3D 그래프 노드가 화면 전체를 채움. react-force-graph-3d로 WebGL 기반 3D 공간 렌더링 | 뷰포트 100% 채움, 노드가 3D 공간에 분포, WebGL 정상 렌더링 |
| 3D 카메라 인터랙션 | 마우스 드래그로 회전(orbit), 스크롤로 줌인/줌아웃, 우클릭 드래그로 패닝 | 회전/줌/패닝 모두 부드럽게 동작 |
| 별빛 노드 효과 | 다크모드에서 노드가 3D 공간에서 glow 효과로 빛남. 클러스터별 고유 색상 적용. Three.js Sprite/Mesh 기반 | glow 효과 적용, 9개 클러스터 색상 구분, 깊이감 표현 |
| 노드 플로팅 애니메이션 | 3D 공간에서 노드가 미세하게 떠다니는 효과 | 자연스러운 3D 움직임 |
| 클릭 시 분할 뷰 전환 | 노드 클릭 시 그래프가 왼쪽(38%)으로 축소 이동, 오른쪽에 콘텐츠 패널 등장. 클릭된 노드로 카메라 자동 포커스 | 전환 애니메이션 smooth, 카메라 이동 자연스러움, 콘텐츠 렌더링 정상 |
| 선택 노드 상태 유지 | 클릭된 노드가 시각적으로 선택 상태 유지 (하이라이트 + 스케일) | 다른 노드 클릭 전까지 유지 |
| 콘텐츠 패널 | MDX 렌더링, 클러스터 뱃지, 태그, 본문 표시 | 기존 MdxRenderer 재사용 |
| 다크모드 토글 | 기존 ThemeToggle 컴포넌트 활용 | 라이트/다크 전환 정상 동작 |
| URL 동기화 | 선택된 노드가 `?node=slug` 쿼리 파라미터로 반영 | 새로고침 시 선택 상태 복원, 공유 가능 |
| 초기 카메라 자동 회전 | 첫 진입 시 카메라가 천천히 자동 회전하여 3D 공간감 연출. 사용자 인터랙션 시 정지 | 자동 회전 동작, 인터랙션 시 즉시 정지 |

## 비기능 요구사항

- **성능**: 3D 그래프 초기 렌더 2초 이내 (50개 노드 기준). LCP 3.0s 미만. WebGL 컨텍스트 정상 생성. 60fps 유지 (노드 50개 이하).
- **보안**: 사용자 입력 없음. XSS 위험 없음.
- **접근성**: 키보드로 노드 탐색 가능 (Tab/Enter). 선택된 노드 aria-current. 테마 토글 aria-label 유지. 3D 캔버스에 aria-label 제공. WebGL 미지원 브라우저에 대한 폴백 메시지 표시.
- **SEO**: `/nodes/[slug]` 정적 페이지 유지 (기존 SEO 이중 경로 전략). 홈의 `?node=slug`는 SPA 탐색용.
- **호환성**: WebGL 2.0 지원 브라우저 필수 (Chrome 56+, Firefox 51+, Safari 15+, Edge 79+).

## 성공 지표 (KPI)

- **첫인상 체류 시간**: 홈 페이지 평균 체류 시간 30초 이상 [Google Analytics]
- **노드 클릭률**: 방문자 중 1개 이상 노드 클릭 비율 50% 이상 [이벤트 트래킹]
- **3D 인터랙션 비율**: 회전/줌 인터랙션을 수행하는 방문자 비율 60% 이상 [이벤트 트래킹]
- **직관적 탐색**: 3개 이상 노드를 탐색하는 사용자 비율 20% 이상 [이벤트 트래킹]
- **포트폴리오 공유**: URL 공유 횟수 (클립보드 복사 이벤트) [이벤트 트래킹]

## 스코프 아웃

이 기능에서 명시적으로 다루지 않는 것:
- `/article`, `/project` 페이지 자체의 구현 (네비게이션 링크만 Phase 2에서 추가)
- VR/AR 모드 지원 (Three.js 기반이지만 현재 스코프 밖)
- 사용자 인증/로그인
- 댓글, 좋아요 등 소셜 기능
- 다국어(i18n) 지원 — 한국어 MVP
- 노드 수정/추가 UI (마크다운 파일 직접 편집으로 유지)
