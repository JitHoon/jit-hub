# 상태 전환 상세

## 상태 다이어그램

```
[초기 화면] ──호버──▶ [리프 펼치기] ──클릭──▶ [분할 뷰]
     ▲                    │                      │
     └────호버 해제────────┘                      │
     ▲                                            │
     └────────────ESC / 닫기 버튼─────────────────┘
```

## 상태 1 → 상태 2: 허브 호버

### 트리거
- 허브 노드에 마우스 진입 (onNodeHover)

### 동작
1. 호버된 허브의 직접 연결 리프 노드 목록 계산
2. 해당 리프 노드들의 `nodeVisibility`를 `true`로 설정
3. 리프 노드가 허브 주변에 애니메이션으로 등장
4. 연결 엣지도 함께 표시

### 되돌리기 (상태 2 → 상태 1)
- 마우스가 허브에서 벗어남 (onNodeHover → null)
- 리프 노드의 `nodeVisibility`를 `false`로 복원
- 애니메이션으로 리프 숨김

### 허브 → 허브 전환
- 다른 허브 호버 시: 이전 리프 접힘 → 새 리프 펼침 (동시)
- 깜빡임 방지를 위해 상태 변경을 한 번의 setState로 배치

## 상태 2 → 상태 3: 노드 클릭

### 트리거
- 허브 또는 리프 노드 클릭 (onNodeClick)

### 동작
1. URL 업데이트: `?node={slug}` 쿼리 파라미터 추가 (shallow routing)
2. 그래프 컨테이너가 좌측으로 축소 (CSS transition)
3. 우측에 본문 패널 슬라이드 인
4. 클릭된 노드의 MDX 본문 렌더링
5. 그래프의 `width` prop 업데이트 → d3 시뮬레이션 재조정

### 본문 패널 구성
- 상단: 노드 제목 + 클러스터 배지 + 난이도 표시
- 본문: MDX 렌더링 결과
- 하단: prerequisites, relatedConcepts 링크 목록
- 관계 유형은 원래 텍스트로 표시 (e.g., "appliedIn", "implements")

### 노드 전환 (상태 3 내부)
- 다른 노드 클릭 → 본문만 전환 (그래프 레이아웃 유지)
- URL 쿼리 파라미터 업데이트
- 그래프에서 새 노드 하이라이트

## 상태 3 → 상태 1: 닫기

### 트리거
- ESC 키 또는 닫기 버튼 클릭

### 동작
1. URL에서 `?node=` 쿼리 파라미터 제거
2. 본문 패널 슬라이드 아웃
3. 그래프 컨테이너가 전체 너비로 복원
4. 리프 노드 숨김 (초기 화면으로)

## 구현 주의사항

### d3 시뮬레이션 관리
- `nodeVisibility` 변경 시 `graphData`를 새 객체로 전달해야 d3가 감지
- 단, 매번 `d3ReheatSimulation`을 호출하면 레이아웃이 흔들림
- `cooldownTicks`와 `warmupTicks`로 안정화 제어

### 리사이즈 처리
- 분할 뷰 전환 시 `width` prop 변경 필요
- ResizeObserver로 컨테이너 크기 추적
- 드래그 핸들로 좌우 비율 조절 시에도 동일

### 모바일 대응
- 768px 이하: 세로 레이아웃 (그래프 위, 본문 아래)
- 호버 → 탭으로 대체 (터치 이벤트)
- 분할 뷰 대신 본문이 그래프 아래에 펼쳐짐
