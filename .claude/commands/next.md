## 지시사항

`progress.md`를 읽고 미완료 작업 목록을 보여준 뒤, 사용자가 선택한 작업 **딱 하나**를 실행하라.

## 프로세스

### 1단계: 작업 선택 (AskUserQuestion 인터랙티브 선택)

1. `progress.md`를 읽어라
2. 모든 미완료 항목(`- [ ]`)을 Phase별로 파악하라
3. **반드시 `AskUserQuestion` 도구를 사용하여** 사용자가 화살표 키로 이동하고 엔터로 선택할 수 있게 하라
4. 미완료 Phase가 여러 개이고, 한 Phase 내 미완료 항목이 4개 이하이면 → 항목을 바로 옵션으로 제시하라
5. 미완료 항목이 4개를 초과하면 → **2단계 선택**을 사용하라:
   - 1차: Phase를 옵션으로 제시 (예: "Phase 3: 콘텐츠 파이프라인", 각 description에 남은 작업 수 표기)
   - 2차: 선택된 Phase 내 미완료 항목을 옵션으로 제시 (4개 초과 시 상위 4개만 표시)
6. 각 옵션의 `label`은 작업명, `description`은 Phase명과 간단한 설명으로 구성하라
7. 사용자가 선택할 때까지 **작업을 시작하지 마라**

### 2단계: 작업 실행

1. 사용자가 선택한 **하나의 작업만** 수행하라
2. 해당 Phase에 명시된 브랜치가 있으면 브랜치 상태를 확인하라
3. 작업을 완료하라
4. 해당 Phase의 검증 기준이 있으면 검증하라

### 3단계: progress.md 업데이트

1. 완료한 항목의 상태 칸을 `[x]`로 변경하라 (테이블 형식 유지)
2. 다른 항목은 절대 수정하지 마라
3. progress.md의 기존 양식(테이블 구조, 섹션 순서)을 변경하지 마라

### 4단계: Phase 완료 시 자동 PR 생성

현재 Phase의 **모든 항목이 완료(`[x]`)** 되었는지 확인하라.

- 모두 완료되었으면 → `/create-pr` 커맨드의 절차를 그대로 따라 실행하라 (`.claude/commands/create-pr.md` 참조)
- Phase가 완료되지 않았으면 → 이 단계를 건너뛰고 5단계로 진행하라

### 5단계: 다음 작업 연속 진행 제안

작업 완료 후 (4단계의 PR 생성이 완료된 후 또는 Phase 미완료 시) **반드시 `AskUserQuestion` 도구를 사용하여** 다음 진행 여부를 물어라:

1. `progress.md`에서 남은 미완료 항목(`- [ ]`)을 다시 파악하라
2. 미완료 항목이 남아 있으면 → 다음 옵션을 제시하라:
   - 같은 Phase 내 다음 미완료 항목들 (있을 경우, 각각 개별 옵션으로)
   - 다른 Phase의 미완료 항목이 있으면 → "다른 Phase 선택..." 옵션 추가
   - "여기서 멈추기" 옵션 (항상 마지막에 포함)
3. 사용자가 특정 항목을 선택하면 → **2단계부터 다시 반복**하라 (선택된 항목을 실행)
4. 사용자가 "다른 Phase 선택..."을 선택하면 → **1단계의 Phase 선택부터 다시 시작**하라
5. 사용자가 "여기서 멈추기"를 선택하면 → 남은 작업 수를 한 줄로 안내하고 종료하라
6. 모든 항목이 완료되었으면 → 축하 메시지와 함께 종료하라

## 절대 규칙

- **한 번에 하나의 `- [ ]` 항목만 수행하라**. 사용자 선택 없이 두 번째 항목으로 넘어가지 마라.
- 작업 완료 후 반드시 4단계(Phase 완료 확인 → PR) → 5단계(연속 진행 제안)를 실행하라.
- Phase의 마지막 항목 완료 시, 검증 + 커밋 + PR 생성까지 자동으로 수행하라.
- CLAUDE.md의 절대 규칙을 준수하라.
